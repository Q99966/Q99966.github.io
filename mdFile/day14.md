# 信号处理

## 什么是信号

- 信号在最早的UNIX系统中即被引入,已有30多年的历史,但只有很小的变化。
- 信号是提供异步事件处理机制的软件中断。
- 这些异步事件可能来自硬件设备,如用户同时按下了Ctrl键和C键,也可能来自系统内核,如试图访问尚未映射的虚拟内存,又或者来自用户进程,如尝试计算整数除以0的表达式
- 进程之间可以相互发送信号,这使信号成为一种进程间通信(Inter-Process
  Communication,IPC)的基本手段
- 信号的异步特性不仅表现为它的产生是异步的,对它的处理同样也是异步的。程序的设计者不可能也不需要精确地预见什么时候触发什么信号,也同样无法预见该信号究竟在什么时候会被处理。一切都在内核的操控下,异步地运行。信号是在软件层面对中断机制的一种模拟

## 异步与同步

### 异步

​	两个程序互不干预的执行，但是有可能会发生冲突。

​	信号中断软件时，是异步事件处理机制，也就是这个终止是不可预知的。比如写个程序，并不能预知运行时会来多少个(ctrl + c)。

### 同步

​	两个程序有协调性的执行。比如有两个进程，进程1和2，进程1执行到某个时间，需要进程2执行到某个位置，才能继续执行，这样互相干预的执行，叫做同步。

## 信号的名称与编号

- 信号是很短的消息,本质就是一个整数,用以区分代表不同事件的不同信号
- 为了便于记忆,在**/usr/include/bits/signum.h**头文件中用一组名字前缀为SIG的宏来标识信号,即为信号的名字。
- 通过kill-1命令可以查看信号
- 一共有62个信号,其中前31个信号为不可靠的非实时信号,后31个为可靠的实时信号

## 信号的处理

- 忽略:什么也不做,SIGKILL(9)和SIGSTOP(19)不能被忽略
- 默认:在没有人为设置的情况,系统缺省的处理行为。（用来恢复默认设置）
- 捕获:接收到信号的进程会暂停执行,转而执行一段事先编写好的处理代码,执行完毕后再从暂停执行的地方继续运行。

### signal

```c
#include <signal.h>
typedef void (*sighandler_t)(int);
// 给函数传的那个int是信号的编号
// 例如这个信号处理函数
void sigfum(int signum)
{
    printf("%d进程：捕获到%d号信号\n",getpid(),signum);
}
sighandler_t signal(int signum, sighandler_t handler);
// 功能:设置调用进程针对特定信号的处理方式
// 参数:signum 信号编号
	// handler 信号的处理方式,可以如下取值
	// SIG_IGN      - 忽略
	// SIG_DFL      - 默认
	// 信号处理的函数指针  - 捕获
// 返回值:成功返回原信号处理方式,如果之前未处理过则返回NULL,失败返回SIG_ERR.（注意指的是相同的信号）
```

​	signal所谓的捕获就是把信号处理函数sigfum的地址交给内核，内核会在后续调用sigfum函数，与注册退出处理函数原理相似。

# 太平间信号

- 如前所述,无论一个进程是正常终止还是异常终止,都会通过系统内核向其
  父进程发送SIGCHLD(17)信号。父进程完全可以在针对SIGCHLD(17)信号的信号处理函数中,异步地回收子进程的僵尸,简洁而又高效

  ```c
  void sigchld (int signum) {
      pid_t pid = wait (NULL);
      if (pid == -1) {
          perror ("wait");
          exit (EXIT_FAILURE);
      }
      printf("%d子进程终止\n",pid);
  }
  ```

## 信号的丢弃

- 在信号处理函数执行期间,如果有相同的多个信号再次到来,只保留一个,其余统统丢弃
  
  可以通过非阻塞处理函数和循环来处理目前死掉的进程
  
  ```c
  void sigchild(int signum)
  {
      while(true) // 在一次信号处理函数中，尽可能多的收尸
      {
          printf("%d进程:捕获到%d号信号\n",
                 getpid(),signum);
          /*
          其他操作
          */
          pid_t pid = waitpid(-1,NULL,WNOHANG);
          if(pid == -1)
          {
  			if(errno == ECHILD)
              {
                  printf("%d进程:没有子进程\n",getpid());
                  break;
              }
              else
              {
  				perror("wait");
                  return;
              }
          }
          else if(pid == 0)
          {
              printf("子进程仍在运行");
          }
          else
          {
              printf("%d进程成功回收",pid);
          }
      }
  }
  ```
  
  
  
- 不同的信号不会丢



# 转储文件

- 发生段错误等并显示（核心已转储）这种请况会将错误转储到一个文件中去，通过gdb corefile xxx , 可以查看详细错误
- 若是文件没有生成，原因是系统限制转储文件的大小为0

### 系统限制

- 系统限制的查看

  ```sh
  ulimit -a
  ```

- 修改系统限制

  ```sh
  # 修改转储文件大小
  ulimit -c 1024
  # 同理若是要修改其他限制，就通过查看命令查看对应的指令就行
  ```
