数据采集与预处理知识点

由两面包夹芝士制作

第1章 概述

1,什么是数据？P1

  • 数据是对客观事物的性质、状态以及相互关系等进行记载的物理符号或这些物理符号的组合，这些符号是可识别的，抽象的。

2,常见数据有哪些？P1

  • 数字，文字，图象，声音等。

3,典型的数据分析过程是什么？P4

  • 数据采集与预处理，数据存储与管理，数据处理与分析，数据可视化。

4,数据采集的三大要点？P6

  • 全面性，多维性，高效性

5,数据采集的主要数据源有哪些？P6

​ 传感器数据，互联网数据，日志文件，企业业务系统数据

6,数据清洗的实现方式和特点？P9

  • 手工清洗：简单但效率低下

  • 自动清洗：可以解决某个特定的问题，但不够灵活。同时这种方法没有充分利用目前数据库提供的强大的数据处理能力。

7,数据清洗中对异常值和缺失值的处理办法。P10

  • 统计值填充、前/后值填充、零值填充。

8,数据清洗的评价标准？P11

  • 数据的可信性、数据的可用性、数据清洗的代价。

9,数据脱敏的原则？P15

  • 保持原有数据特征、保持数据的一致性、保持业务规则的关联性、多次脱敏的数据一致性。

第2章 大数据实验环境搭建

1,Python的优点？P19

  • 语言简单，开源、免费，面向对象，跨平台，强大的生态系统。

2,Python的基本数据类型？P22

  • 数字和字符串。

3,什么是JDK？P27

  • Java 开发工具包(Java Development Kit, JDK)是整个Java的核 心，包括了Java 运行环境( Java Runtime Environment )、Java 工具和Java 基础类库等。

4,什么是数据库？P28

  • 数据库是一种主流的数据存储和管控技术，指的是以一定方式存储在一起，能为多个用户共享、具有尽可能小的冗余度、与应用程序彼此独立的数据集。

5,什么是SQL？P29

  • sql是关系数据库的标准语言，也是一个通用的功能极强的关系数据库语言，其功能包括查询、数据库创建、数据库数据的插入和修改、数据库安全性与完整性定义等。

6,什么是HDFS？P41

  • HDFS是Hadoop项目的两大核心之一，是针对谷歌文件系统的开源实现。

7,Hadoop的安装模式有哪些？P42

  • 单机模式，伪分布式模式，分布式模式

第3章 网络数据采集

1,什么是网络爬虫？P49

  • 网络爬虫是一个自动提取网页的程序，它为搜索引擎从万维网上下载网页，是搜索引擎的重要组成部分。

2,网络爬虫的工作原理？P49

  • 网络爬虫从一个或若干个初始网页的URL 开始，获得初始网页上的URL,在抓取网页的过程中，不断从当前页面上抽取新的URL放入队列，直到满足系统的一定停止条件。

3,网路爬虫的类型有哪些？P50

  • 通用网络爬虫，聚焦网络爬虫，增量式网络爬虫，深层网络爬虫。

4,反爬机制的优缺点？P51

  • 优点：可以保护企业网站和网站数据

  • 缺点：如果反爬机制过于严格，真正的用户的请求会被误当网络爬虫而被拒绝。增加网站研发的成本。

5,什么是HTTP？P52

  • HTTP是由万维网协会和互联网工程任务组共同制定的规范。HTTP用于从网络传输超文本数据到本地浏览器，它能保证高效而准确的传送超文本内容。

6,HTTP的请求过程是什么？P52

  • （1）用户在浏览器中输入网址，浏览器向Web服务器发起请求。

  • （2）Web服务器接收用户访问请求，处理请求，产生响应（即把处理结果以HTML形式发送给浏览器）。

  • （3）浏览器接受来自Web服务器的HTML内容，进行渲染以后展示给用户。

第4章 分布式消息系统Kafka

1,Kafka的应用场景有哪些？P89

  • 日志收集，消息系统，用户活动跟踪，运营指标，流式处理。

2,什么是Kafka？P88

  • Kafka是由LinkedIn公司开发的一种高吞吐量的分布式消息订阅分发系统，用户通过Kafka系统可以发布大量的消息，同时也能实时订阅和消费信息。

3,kafka在大数据生态中的作用和优势？P90

  • 在企业的大数据生态系统中，可以把Kafka作为数据交换枢纽，不同类型的分布式系统统一接入Kafka，实现和Hadoop各个组件之间的不同类型数据的实时高效交换，较好的满足各种企业的应用需求。同时也可以很好的解决不同系统之间的数据产生/消费速率不同的问题。

第5章 日志采集系统Flume

1,什么是Flume？P102

  • 是Cloudera提供的一个高可用，高可靠，分布式的海量日志采集、聚合和传输系统，其支持在日志系统中定制各类数据发送方，用于采集数据。同时，其提供对数据进行简单处理，并写到各种数据接受方（可定制）的能力。

2,Flume的核心组件有哪些？P102

  • 数据源、数据通道和数据槽。

第6章 数据仓库中的数据集成

1,什么是数据仓库？P1138

  • 是一个面向主题、集成、相对稳定、反映历史变化的数据集，用于支持管理决策。

2,数据仓库和3数据库有什么区别？P119

  • 数据库是面向事物设计，数据仓库是面向主题设计。

3,什么是ETL？P122

  • 将业务系统的数据抽取、转换之后加载到数据仓库的过程，目的是将企业中的分散、零乱、标准不统一的数据整合到一起，为企业的决策提供分析依据。
