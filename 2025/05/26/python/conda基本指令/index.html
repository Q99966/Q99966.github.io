conda基本指令

总结借鉴Anaconda conda常用命令：从入门到精通_conda list-CSDN博客

常用命令

  • conda list //查看安装了哪些包
  • conda env list 或者 conda info -e //查看当前存在那些虚拟环境
  • conda update conda // 更新当前conda
  • conda update Anaconda // 更新Anaconda整体
  • conda —version //查询conda 版本
  • conda -h // 查询conda的命令
  • conda create —help // 查询某个命令的帮助

查看conda的环境配置

  • conda config —show

设置镜像

  • conda config —add channels 链接 // 设置镜像源
  • 设置bioconda
    • conda config —add channels bioconda
    • conda config —add channels conda-forge
  • 设置搜索时显示通道地址
    • conda config —set show_channel_urls yes

管理虚拟环境

  • 创建虚拟环境

    • conda create -n env_name python=3.8

  • 创建虚拟环境的同时安装必要的包

    • conda create -n env_name numpy matplotlib python=3.8

  • 查看有哪些环境

    • conda env list
    • conda info -e
    • conda info —envs

  • 激活虚拟环境

    • conda activate env_name

  • 退出虚拟环境

    • conda activate
    • conda deactivate

  • 删除虚拟环境

    • conda remove —name env_name —all
    • 只删除虚拟环境中的某个或者某些包则是
      • conda remove — name env_name package_name

  • 导出环境

    • conda env export —name myenv > myenv.yml // 获得环境中的所有配置
    • conda env create -f myenv.yml // 重新还原环境
    • 导出的文件就在所处目录下

    • 如上图 导出文件在C:\Users\chenzhihan 下

包的管理

  • 查询包的安装情况

    • conda list // 查看当前环境中安装了哪些包
    • 查询当前Anaconda repository 中是否有你想要安装的包
      • conda search package_name // 必须联网

  • 查询是否有安装某个包

    • conda list pkgname
    • conda list pkgname* // 模糊查找

  • 包的安装和更新

    • conda install package_name
    • conda install numpy=0.20.3
    • 更新
      • conda update numpy

  • conda卸载包

    • conda uninstall package_name // 同时卸载依赖包
    • conda uninstall package_name —force // 不会删除依赖包 — 不建议

  • 清理anaconda缓存

    • conda clean -p // 删除没有用的包
    • conda clean -t // 删除tar打包
    • 删除所有的安装包及cache(索引缓存、锁定文件、未使用过的包和tar包)
      • conda clean -y -all
    • 查看clean命令
      • conda clean -h

Python版本的管理

  • 将版本变更到指定版本
    • conda install python=3.5
    • python —version
  • 将python版本更新到最新版本
    • conda update python

Conda install vs pip install

区别

  1. conda可以管理非python包，pip只能管理python包。

  2. conda自己可以用来创建环境，pip不能，需要依赖virtualenv之类的。

  3. conda安装的包是编译好的二进制文件，安装包文件过程中会自动安装依赖包；pip安装的包是wheel或源码，装过程中不会去支持python语言之外的依赖项。

  4. conda安装的包会统一下载到一个目录文件中，当环境B需要下载的包，之前其他环境安装过，就只需要把之间下载的文件复制到环境B中，下载一次多次安装。pip是直接下载到对应环境中。

  5. conda只能在conda管理的环境中使用，例如比如conda所创建的虚环境中使用。pip可以在任何环境中使用，在conda创建的环境 中使用pip命令，需要先安装pip（conda install pip ），然后可以 环境A 中使用pip 。conda 安装的包，pip可以卸载，但不能卸载依赖包，pip安装的包，只能用pip卸载。

  • 能否混用

    • 混用容易导致库的依赖关系出现混乱
    • 只有在conda install搞不定时才使用pip intall。 而且，最后使用虚拟环境进行环境隔离。

  • 库都安装在哪

    • conda install xxx
    • anaconda3/pkgs // 我的anaconda在c : ProgramData
      • 这样的好处就是，当在某个环境下已经下载好了某个库，再在另一个环境中还需要这个库时，就可以直接从pkgs目录下将该库复制至新环境而不用重复下载。
    • pip install xxx
      • 分两种情况，一种情况就是当前conda环境的python是conda安装的，和系统的不一样，那么xxx会被安装到anaconda3/envs/current_env/lib/python3.x/site-packages文件夹中，如果当前conda环境用的是系统的python，那么xxx会通常会被安装到~/.local/lib/python3.x/site-packages文件夹中

  • 如何判断conda中某个包是通过conda还是pip安装的？

    执行conda list , 用pip安装的包显示的build项目为pypi
